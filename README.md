# Code2Context

A Python utility to generate a compact, LLM-friendly context file from your app's source code. It scans directories, filters files by extension, summarizes frontend/UI code, and outputs a delimited text file for LLMs.

## Features
- **Configurable**: Use command-line args or JSON config.
- **Summarization**: Reduces token count by summarizing frontend files (e.g., React `.tsx`).
- **Flexible**: Supports multiple languages (Python, JS/TS, Java, C++, etc.).
- **Compact Output**: Uses `=== filepath ===` delimiters, optimized for LLMs.

## Installation
1. Clone: `git clone https://github.com/shawnmarck/code2context.git && cd code2context`
2. Install: `pip install -r requirements.txt` (none beyond Python stdlib currently).

## Usage
Run with defaults: `python code2context.py`  
Scans `./src` and `./app`, outputs to `app_context.txt`.

### Options
- `--output <file>`: Output file (default: `app_context.txt`).
- `--config <file>`: JSON config (default: `context_config.json`).
- `--purpose <text>`: App purpose.
- `--tech <text>`: Tech stack.
- `--dirs <dir1 dir2 ...>`: Directories to scan.
- `--exts <ext1 ext2 ...>`: File extensions.

Example: 
```bash
python code2context.py --output myapp.txt --purpose "E-commerce" --tech "React/Flask" --dirs ./frontend ./backend --exts .py .tsx
```

### Config File
Create `context_config.json`:
```json
{
    "output_file": "myapp.txt",
    "include_dirs": ["./frontend", "./backend"],
    "exclude_patterns": ["node_modules", "__pycache__", ".git"],
    "include_extensions": [".py", ".js", ".ts"],
    "summarize_frontend": true,
    "frontend_extensions": [".tsx", ".jsx"],
    "min_file_size": 50
}
```

Run: 
```bash
python code2context.py --config context_config.json
```

## Output Format

```
# App Context | Mar 14, 2025
Purpose: E-commerce
Tech: React/Flask
Generated by code2context.py

=== backend/app/models.py ===
class Product: ...

=== frontend/src/pages/Cart.tsx ===
useState(...); useEffect(...);
```

## Customization
- **Add Languages**: Extend include_extensions.
- **Adjust Summarization**: Modify summarize_frontend() keywords.
- **Exclusions**: Add to exclude_patterns.

## Token Efficiency
 - **Reduction**: Summarizes frontend code and skips small files, reducing token count significantly vs. a flat file dump.
 - **Examples**:
    - **SamplePythonApp**: Original 225K chars (50K–70K tokens) reduced to 70K chars (15K–20K tokens), a ~65–70% reduction.
    - **Generic** App: A 500K char codebase (125K tokens flat) can shrink to ~150K chars (35K–40K tokens), a ~68–72% reduction.
- Fits within LLM context windows (e.g., Claude 3’s 200K tokens) with room for prompts.
